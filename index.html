Install

Install dependencies with Yarn.

$ yarn

You can also use npm.

$ npm install

Build

The project is built in TypeScript.

$ npm run build

Build output will be available in dist/.

Start

The project will need to be built before you can start the server.

$ npm start

Will read the environment variables HOST and PORT when starting the API server. These values default to 127.0.0.1 and 8080 respectively. The server will then be available at http://127.0.0.1:8080/.

Test

Code is linted with TSLint.

Unit tests are run with AVA. To test the latest code changes make sure to build the project first.

The test suite will connect to your configured development database so make sure it is up to date and seeded before running.

$ npm test

Docs

Documentation is generated with TypeDoc.

$ npm run docs

Docs output will be available in docs/.

Database

Databases are provisioned with Knex. Environments are configured in db/knexfile.js. There are two environments set up:

  • development (default)

    A simple SQLite database for easy development and testing.

  • production

    A PostgreSQL database that is configured via the CONNECTION_STRING environment variable.

To connect to a production database you will need to run the following commands with the NODE_ENV environment variable set to production.

Update database

$ npm run latest

Rollback database update

$ npm run rollback

Create new migration

$ npm run make-migration

Creates a new migration file in db/migrations.

Seed database

$ npm run seed
